# Data Model: Frontend Pink Theme with CRUD Functionality

## Overview
This document defines the data models for the frontend application, based on the entities identified in the feature specification. These models will be implemented in TypeScript for type safety and consistency.

## User Entity

### User
Represents a registered user in the system.

**Fields**:
- `id`: string - Unique identifier for the user
- `name`: string - User's display name
- `email`: string - User's email address (unique)
- `createdAt`: Date - Timestamp when the user account was created
- `updatedAt`: Date - Timestamp when the user account was last updated

**Validation Rules**:
- `name`: Required, minimum 2 characters, maximum 50 characters
- `email`: Required, valid email format, maximum 100 characters
- `id`: Auto-generated by backend system

## Task Entity

### Task
Represents a user's task with title, description, completion status, and timestamps.

**Fields**:
- `id`: string - Unique identifier for the task
- `title`: string - Task title
- `description`: string - Detailed description of the task (optional)
- `completed`: boolean - Whether the task is completed or not
- `userId`: string - ID of the user who owns this task
- `createdAt`: Date - Timestamp when the task was created
- `updatedAt`: Date - Timestamp when the task was last updated
- `completedAt`: Date | null - Timestamp when the task was marked as completed (null if not completed)

**Validation Rules**:
- `title`: Required, minimum 1 character, maximum 100 characters
- `description`: Optional, maximum 500 characters
- `completed`: Boolean, defaults to false
- `userId`: Required, must match authenticated user's ID
- `id`: Auto-generated by backend system

## Agent Entity

### Agent
Represents an available agent with skills and capabilities.

**Fields**:
- `id`: string - Unique identifier for the agent
- `name`: string - Agent's display name
- `description`: string - Description of the agent's capabilities
- `createdAt`: Date - Timestamp when the agent was registered
- `updatedAt`: Date - Timestamp when the agent was last updated

**Validation Rules**:
- `name`: Required, minimum 2 characters, maximum 50 characters
- `description`: Required, minimum 10 characters, maximum 200 characters
- `id`: Auto-generated by backend system

## Skill Entity

### Skill
Represents a specific skill or capability that agents may possess.

**Fields**:
- `id`: string - Unique identifier for the skill
- `name`: string - Name of the skill
- `description`: string - Description of the skill
- `categoryId`: string - Category or type of the skill
- `createdAt`: Date - Timestamp when the skill was registered
- `updatedAt`: Date - Timestamp when the skill was last updated

**Validation Rules**:
- `name`: Required, minimum 2 characters, maximum 50 characters
- `description`: Required, minimum 10 characters, maximum 200 characters
- `categoryId`: Required, maximum 50 characters
- `id`: Auto-generated by backend system

## API Response Models

### AuthResponse
Response model for authentication requests.

**Fields**:
- `user`: User - The authenticated user object
- `token`: string - JWT token for subsequent requests
- `expiresAt`: Date - Expiration timestamp for the token

### TaskListResponse
Response model for fetching user's tasks.

**Fields**:
- `tasks`: Task[] - Array of tasks belonging to the user
- `total`: number - Total count of tasks

### AgentListResponse
Response model for fetching agents.

**Fields**:
- `agents`: Agent[] - Array of available agents
- `total`: number - Total count of agents

### SkillListResponse
Response model for fetching skills.

**Fields**:
- `skills`: Skill[] - Array of available skills
- `total`: number - Total count of skills

## State Management Models

### UI State Models
Models used for managing frontend state.

**UserState**:
- `user`: User | null - Currently authenticated user
- `token`: string | null - JWT token
- `loading`: boolean - Whether authentication is in progress
- `error`: string | null - Any authentication error

**TaskState**:
- `tasks`: Task[] - List of user's tasks
- `loading`: boolean - Whether tasks are being loaded
- `error`: string | null - Any task-related error
- `currentTask`: Task | null - Currently selected task (for editing)

**AgentSkillState**:
- `agents`: Agent[] - List of available agents
- `skills`: Skill[] - List of available skills
- `loading`: boolean - Whether agents/skills are being loaded
- `error`: string | null - Any agent/skill-related error
